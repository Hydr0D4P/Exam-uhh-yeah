Accessibility

Before even building, and throughout

Before even writing a single line of code, there are a couple of things I tried my best to keep in mind.

Firstly, as a person with a background in visual design, colour pallets are super cool. I spent a fair bit of time trying and testing different colours. Originally the website was a lot more cyan, but I felt it contrasted poorly with the yellow colour that OsloMet uses. When looking for logo elements for OsloMet I stumbled upon this site, and it was just following the official rules from there. I essentially went for a Dark mode OsloMet look instead of my original dark cold sleek look. For the text I use a simple Dark on Bright, Bright on Dark rule, considering luminosity and contrast over hue, to assist colour blind people.

For the blind or weak sighted, I wanted to make sure that the text is both large enough, and that the general structure of my code is easy to understand for a screen reader. I, therefore, have a “skip to content”, so that people using screen readers and tabbing don't have to interact with the navigation menu every time they land on a page. It doesn’t show up unless you tab into it. It also used to have animation but I felt it unnecessary. I also made sure that all my text is at least 16px, even at the lowest resolutions. The text on the images can get small at times, but they have pretty lengthy descriptions if that is needed.

I knew I wanted some animations from the get-go, but I wanted them a) not to be distracting, and b) not to interfere with the normal flow. Other than the “Fun” page, that remains consistent. I originally wanted the animation on the text to be applied to h1, but it became too noisy with it there.

The site also changes slightly depending on what resolution you are using it at, so that mobile users or half window people can still use the website as intended.

Post-Construction

After running the site through a couple of automated HTML and CSS validators, there were essentially no issues. I was originally using the bold element and changed that too strong, but other than the “fun” page, things looked good.

I ended up doing another pass of my image descriptions, removing some bloat, and adding some detail. I also had a div without any purpose around my containers that also got removed. Some CSS was moved around as well, and some lines were combined (a and strong, wrapper p and wrapper h2).

The Funny.css, the CSS that controls has a lot of issues, especially my use of webkits, but I couldn't get it to work otherwise. Given the extra time that could be solved, but it wasn’t a primary focus.
